# Telegram 双向聊天机器人项目说明

## 项目定位

面向 Telegram 私聊场景的双向聊天中转系统，适合作为客服入口、咨询入口或工单接待机器人使用。

部署后：用户直接与机器人私聊，管理员在私聊窗口或超级群话题中处理每条咨询。

## 核心能力

**双向聊天** — 用户消息转发给管理员，管理员回复后回传给用户。

**话题模式** — `TOPIC_MODE=true` 时，每个用户在管理员超级群中自动分配独立话题，适合多用户并发管理。

**首次验证** — `USER_VERIFICATION=true` 时，新用户需先通过验证才能进入正常会话。

**风控管理** — 黑名单、信任名单、关键词过滤、多管理员授权。

**后台面板** — 独立 Vue 3 前端（Cloudflare Pages），提供用户管理、配置维护、历史消息查看等完整界面。

## 技术架构

| 组件 | 技术 | 职责 |
|---|---|---|
| `worker.js` | Cloudflare Workers | Webhook 接收、消息转发、后台 API |
| `BOT_KV` | Cloudflare KV | 用户状态、话题映射、黑白名单、授权 |
| `DB` | Cloudflare D1 | 聊天历史记录（可选） |
| `admin-panel/` | Vue 3 + Vite | 后台管理界面，部署到 Pages |
| `electron-app/` | Electron | 桌面部署客户端 |

## 部署方式

所有部署操作通过 `electron-app/` 桌面客户端完成，无需手动执行命令行。

详见 `README.md`。

## 适用场景

- Telegram 私聊客服机器人
- 频道/社群咨询入口
- 个人业务咨询机器人
- 小团队售后或运营中转系统

## 安全建议

- 不要把 `BOT_TOKEN` 提交到公开仓库
- 首次登录后台后立即修改临时密码
- 正式环境建议为后台单独绑定域名
- 建议配置 `WEBHOOK_SECRET` 和 `ADMIN_API_KEY`

## 可扩展方向

- 带 AI 自动回复的客服版本
- 带标签与备注的 CRM 版本
- 带审计日志的团队协作版本
