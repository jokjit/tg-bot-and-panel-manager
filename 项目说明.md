# Telegram 双向聊天机器人项目说明

## 一、项目定位

这是一个面向 Telegram 私聊场景的双向聊天中转系统，适合把机器人当成“客服入口”或“咨询入口”来使用。

项目部署后，普通用户只需要和机器人私聊；管理员则可以在管理员私聊窗口，或者管理员超级群的话题中，直接处理每个用户的咨询。

相比纯命令式机器人，这个版本更偏向“运营可用”：

- 支持用户与管理员之间的双向沟通
- 支持管理员授权和多管理员协作
- 支持黑名单、信任名单、关键词风控
- 支持首次验证
- 支持后台管理界面
- 支持 Cloudflare Workers / KV / Pages 的低成本部署

## 二、核心能力

### 1. 双向聊天

- 用户发送消息给机器人
- 机器人把消息转发给管理员
- 管理员回复后，消息回传给原用户

### 2. 话题模式

当 `TOPIC_MODE=true` 且 `ADMIN_CHAT_ID` 指向已开启 Topics 的超级群时：

- 每个用户会自动分配一个独立话题
- 用户的后续消息会持续进入同一话题
- 管理员可以把每个用户当作一条独立工单来处理

### 3. 首次验证

当 `USER_VERIFICATION=true` 且绑定 `BOT_KV` 后：

- 用户第一次私聊时不会立即进入管理员侧
- 需要先通过验证
- 验证通过后才进入正常会话

### 4. 运营管理能力

- 黑名单：阻止指定用户继续联系
- 信任名单：跳过部分风控逻辑
- 关键词过滤：适合基础风控和垃圾消息阻断
- 管理员授权：支持多人协作

### 5. 浏览器后台

项目提供两种后台使用方式：

- Worker 内置 `/admin` 页面
- 独立 `admin-panel` 前端页面

独立前端更适合正式运营，因为界面更完整，也更适合 Cloudflare Pages 自动部署。

## 三、技术结构

### Worker 层

负责：

- 接收 Telegram Webhook
- 转发消息
- 执行管理员命令
- 输出状态页和后台 API

核心文件：`worker.js`

### 数据层

负责：

- 保存用户档案
- 保存话题映射
- 保存黑名单 / 信任名单 / 管理员授权
- 保存首次验证状态

核心绑定：`BOT_KV`

### 前端后台

负责：

- 登录
- 仪表盘
- 用户与权限管理
- 系统配置维护

核心目录：`admin-panel/`

## 四、适合的使用场景

- Telegram 私聊客服机器人
- 频道广告投放咨询入口
- 个人业务咨询机器人
- 小团队售后或运营中转系统
- 需要轻量级客服后台的项目

## 五、部署建议

### 推荐组合

- Worker：Cloudflare Workers
- 状态存储：Cloudflare KV
- 后台前端：Cloudflare Pages

### 推荐域名

- Worker：`https://tg.example.com`
- 后台：`https://tg-admin.example.com`

## 六、部署步骤概览

1. 创建 Telegram Bot，拿到 `BOT_TOKEN`
2. 创建管理员群，拿到 `ADMIN_CHAT_ID`
3. 把项目推送到 GitHub 仓库
4. 配置 `wrangler.toml`
5. 创建 `BOT_KV`
6. 通过 `wrangler secret put` 配置敏感变量
7. 部署 Worker
8. 设置 Webhook
9. 部署 `admin-panel`
10. 绑定自定义域名并联调

补充说明：

- `BOT_TOKEN` 与 `ADMIN_CHAT_ID` 是机器人正常工作的必要项
- 这两个值既可以在 Worker 的环境变量 / Secrets 中配置，也可以在后台登录后写入 KV
- 当 `BOT_TOKEN` 与 `ADMIN_CHAT_ID` 可用后，系统会自动生成 1 小时有效的首次临时密码，并发送到 `ADMIN_CHAT_ID`
- 用户首次登录后会被强制跳转到改密页，新密码保存到 KV 中长期生效，直到用户再次修改
- 如果没有配置自定义 `PUBLIC_BASE_URL`，Webhook 会默认使用 Cloudflare 分配的 Worker 域名
- `ADMIN_IDS`、`WEBHOOK_SECRET`、`ADMIN_API_KEY`、`PUBLIC_BASE_URL`、`ADMIN_PANEL_URL` 这类扩展配置不必在首次部署时填满，后续可以在面板中继续维护
- 如果首次临时密码消息没有收到，可在管理员 Telegram 会话中使用 `/panelpass` 重发，或用 `/panelreset` 生成新的临时密码

更详细的命令和流程请直接看根目录 `README.md`。

## 七、一键部署

项目已适配 Cloudflare 官方的一键部署按钮格式：

```md
[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/jokjit/tg-bot-and-panel-manager)
```

注意：

- 该按钮只覆盖 Worker 部分
- 独立后台前端仍建议走 Cloudflare Pages

## 八、安全建议

- 不要把 `BOT_TOKEN` 提交到公开仓库
- 建议在首次登录后立即修改面板永久密码
- 正式环境建议为后台单独绑定域名
